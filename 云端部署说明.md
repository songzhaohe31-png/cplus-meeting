# CPLUS预约会议系统 — 云端部署（老板/客户随时扫码）

## 为什么用云端？

| | 挂你私人电脑公网 | **云服务器（正确）** |
|--|------------------|----------------------|
| 电脑关机 | 不能约 | **仍能约** |
| 私人文件 | 有暴露风险 | **不在服务器上** |
| 老板/客户 | 4G/5G 可约 | **4G/5G 可约** |
| 安全性 | 差 | **只有预约系统 + 访问码** |

---

## 方案：Render 免费部署（约 10 分钟）

### 准备
- 一个邮箱注册 [https://render.com](https://render.com)（免费）
- 本文件夹：`桌面/CPLUS会议室预约系统`

### 步骤 A：把代码放到 GitHub（Render 需要从仓库部署）

1. 打开 https://github.com 注册/登录  
2. New repository → 名称例如 `cplus-meeting` → 创建（可 Private）  
3. 在终端执行（把 `你的用户名` 换成自己的）：

```bash
cd ~/Desktop/CPLUS会议室预约系统
git init
git add server.py static Dockerfile render.yaml requirements.txt
git commit -m "CPLUS meeting room booking"
git branch -M main
git remote add origin https://github.com/你的用户名/cplus-meeting.git
git push -u origin main
```

（若提示登录，按 GitHub 提示完成即可。）

### 步骤 B：在 Render 创建服务

1. 登录 https://dashboard.render.com  
2. **New +** → **Web Service**  
3. 连接刚才的 GitHub 仓库 `cplus-meeting`  
4. 设置：  
   - **Name**: `cplus-meeting`  
   - **Runtime**: Docker  
   - **Instance type**: Free  
5. **Environment** 环境变量：  
   - `ACCESS_PIN` = 你设的访问码，例如 `2580`（告诉老板/同事即可）  
   - `DATA_DIR` = `/data`  
6. **Disk**（重要，否则重启丢数据）：  
   - Add Disk → Mount path `/data` → 1 GB  
7. Create Web Service → 等待部署（首次约 5–10 分钟）  
8. 部署成功后，Render 给你一个地址，类似：  
   `https://cplus-meeting-xxxx.onrender.com`  

### 步骤 C：生成扫码海报

```bash
cd ~/Desktop/CPLUS会议室预约系统
python3 生成云端二维码.py https://cplus-meeting-xxxx.onrender.com
```

桌面出现 **`CPLUS预约会议系统.png`**  
- 海报上只写系统名，不写域名  
- 发给老板/客户，或打印贴会议室门口  

### 步骤 D：使用

1. 扫码打开  
2. 输入访问码（你在 Render 设的 `ACCESS_PIN`）  
3. 选会议室、时间、人数 → 预约  
4. **你的电脑可以关机**，系统照常运行
